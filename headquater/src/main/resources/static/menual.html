1. W-Hive 시작

1.1. 회원가입

W-Hive 회원 가입시 필요한 정보를 입력해야 합니다.

        (1) 아이디, 중복 확인

        (2) 이메일

(3) 본인인증 : 사용자 인증시 email로 전달한 번호

        (4) 이름

        (5) 비밀번호

        (6) 비밀번호 확인

위의 내용을 입력을 완료하고 회원가입 버튼을 클릭하면 가입 완료가 됩니다.

가입이 완료 된 이후 메일이 발송됩니다.

해당 메일에 주소로 접속하면 W-Hive 실행 화면으로 이동합니다.

        (1) ID

        (2) Password

        (3) Login 버튼

        - 버튼은 다음과 같습니다.

        (4) 아이디 저장

        (5) 회원가입

        (6) 비밀번호 재설정

        (7) 아이디 찾기

내용과 같이 화면이동 버튼이 있습니다.

아이디, 패스워드를 입력하고 로그인 버튼을 누르면 W-Hive 실행 화면으로 이동합니다.

1.3. 아이디, 비밀번호 재설정

아이디를 분실했을시에는 W-Hive 서비스 가입시 입력했던 E-Mail 정보를 입력하시고 전송을 눌러주세요.

비밀번호 재설정을 원하시는 경우에는 W-Hive 서비스 가입시 입력했던 ID, E-Mail 정보를 입력하시고 전송 버튼을 눌러주세요.

해당 이메일 정보로 발송되면 임시 비밀번호를 받게 됩니다.

임시비밀번호로 로그인 하시고 사용자정보 메뉴로 이동하셔서 비밀번호를 변경하시기 바랍니다.

2. W-Hive Project 환경 설정

2.1. Workspace 관리

Workspace 는 Project를 그룹으로 관리하는 개체입니다. 상단의 화면은 Workspcae 관리 화면 초기 상태 입니다.

2.1.1 workspace 생성

Workspcae 생성을 하고자 할때 빨간색 박스안에 있는 내용과 같이 Workspace 이름을 입력하고 생성 버튼을 누르면 하단에 결과 처럼 Workspace List 에 조회가 되면서 리스트에 데이터를 볼수가 있습니다.

2.1.2 Workspace 수정

Workspace 수정 시 가능한 기능은 Workspace 이름, 활성 여부 체크 입니다.

  • Workspace : 앱 프로젝트를 관리하기위한 공간 이며 수정시, 더블클릭하고 변경하고 싶은 이름으로 수정하면 됩니다.
  • 활성여부 : Workspace 사용상태 여부 확인이 가능하며  실행 화면에서 Workspace 를 보여지지 않기를 원하거나 쓰지 않는다면 체크를 해제하여 비활성화를 할수 있습니다.
  • Workspace 이름 수정 및 활성여부 체크를 하고나서 파란색 수정 버튼을 클릭 하면 변경사항을 저장할 수 있습니다. 수정 버튼을 누르지 않으면 변경사항은 적용 되지 않습니다.

2.1.3 Workspace 삭제

Workspace 삭제를 하기 원하시면 노란색 delete 버튼을 누르시고 안내 창이 나오면 확인을 누릅니다.

삭제를 더 신중히 하기 위해서 삭제하고자 하는 Workspace 이름을 입력해야합니다. Workspace 이름을 동일하게 입력하고 삭제 버튼을 누르면 Workspace 삭제가 됩니다.

2.2.2 iOS

코드사이닝(Code Signing)

“애플만이 자신들의 하드웨어에서 소프트웨어(앱, 프로그램 등)를 동작하도록 허락하고 있다.”

iOS앱이 실행될때마다 애플로부터 인증여부를 확인하며 앱을 실행할 수 있는 권한이 주어졌는지 확인합니다. 애플만 앱을 실행할 수 있는 권한을 가지기 때문에 앱 개발자들은 iOS Developer Program의 개발자 센터에서 애플인증서를 발급받아 애플 대신 앱을 실행할 권한을 부여 받습니다.

Apple 공식 코드사이닝 가이드

https://help.apple.com/xcode/mac/current/#/dev60b6fbbc7

코드서명에 사용되는 애플 개발자 인증서와 프로비저닝 프로파일을 생성하는 방법과 appstore 배포시 필요한 appstore connect key(*.p8)키 발급방법을 알아보도록 하겠습니다.

준비사항

  • 애플 계정
  • 애플 개발자 프로그램 (개인(AppStore) or 기업(Enterprise))
  • Bundle Identifier (앱 구분자)
  • macOS 장비

2.2.2.1 인증서발급

인증서발급 순서는 아래와 같습니다.

CSR파일생성 > 애플 개발자센터에 CSR파일 업로드 > 인증서 발급

CSR(Certificate Signing Request)파일 :  개발자 인증 요청서로 이 파일은 이름, 이메일, 공개키를 포함하고 있으며, 개인키로 사이닝됩니다.

CSR파일 생성

  • macOS기기에서 “키체인 접근"앱을 실행합니다.
  • 키체인접근 > 인증서지원 > 인증기관에 인증서 요청 … 메뉴를 실행합니다.

  • 정보입력 및 요청항목에 “디스크에 저장됨” 선택

  • 애플에 업로드할 CSR파일이 생성됩니다.

인증서 생성

  • 애플 개발자 사이트에 접속합니다. (https://developer.apple.com)

  • 좌측메뉴에 Certificates, IDS, & Profiles을 선택합니다.

  • 좌측메뉴에 Certificates 메뉴를 선택하고 Certificates 옆에 +버튼을 클릭합니다.

  • iOS App Development 또는 iOS Distribution 선택합니다.

용도에 따라 선택

iOS App Development : 개발용 인증서

iOS Distribution : App Store 용 인증서 (해당 인증서로 TestFlight 등록)

  • CSR 파일 업로드

  • 발급받은 인증서 다운로드 후 더블 클릭하여 실행하면 “키체인 접근”앱에 저장됩니다.

  • “키체인 접근”앱에 저장된 개발자인증서를 “내보내기”로 PKCS#12형식으로 p12파일을 생성합니다. 파일이름과 파일에 부여할 비밀번호를 입력하면 생성됩니다.

이미 발급된 애플 개발자 인증서가 있다면 인증서를 저장하고 있는 “키체인 접근”앱에서 PKCS#12(*.p12)형식으로 export하여 사용합니다.

2.2.2.2 프로비저닝 프로파일 생성

프로비저닝 프로파일(Provisioning Profile)은 app id, certificate, device 정보를 가지고 있어, iOS기기들과 인증서를 연결 해 주는 역할을 합니다.

프로비저닝 프로파일 생성방법

프로비저닝 생성 순서

App ID 생성 > Device 등록 > Provisioning Profile 생성

App ID 생성

  • Bundle Identifiers의 등록되어 있는 AppId 를 가진 앱만 배포/테스트 가능
  • Bundle Identifiers 메뉴 선택 후 +버튼 선택

  • App IDs 선택

  • App 선택

  • Description, Bundle Identifier 입력, Capabilities 선택 (push 등 필요시 선택)

  • Continue 버튼을 눌러 AppID 생성

Device등록 (배포 프로비저닝 프로파일 생략가능)

  • 좌측 Devices 메뉴 선택 후 화면에 +버튼을 클릭하여 기기추가

  • DeviceName, Device ID(UDID) 입력
  • Device ID : UDID 값 입력
  • DeviceName : 등록할 기기이름

UDID 확인방법

  1. mac장비가 있는 경우 케이블로 iphone과 연결시 Finder를 통해서 UDID 확인이 가능 합니다. (UDID항목이 안보인다면 빨간박스부분을 클릭하시면 내용이 변경됩니다.)

  1. iphone만있는경우 safari에서 https://get.udid.io/ 사이트를 방문하여 확인가능합니다.

Provisioning Profile 생성

  • 좌측 Profile메뉴에서 화면이동후 +버튼 선택

  • 용도에 맞는 항목선택
  • iOS App Development(개발용)
  • AppStore(TestFilght, AppStore)
  • In House (내부배포용)

Enterprise 계정에 경우 In House를 선택할 수 있습니다.

  • 생성한 App ID 선택

  • 생성한 Certificate 선택

  • Device 선택(배포용 프로파일 발급시 없는 항목)

  • Profile Name 입력

  • 프로파일 발급이후 다운로드(*.mobileprovision)

2.2.2.3 App Store Connect Key 발급(*.p8)

API Key를 이용해 Apple 서비스를 구성 및 인증 하는데 사용된다.

공식 가이드

https://developer.apple.com/documentation/appstoreconnectapi/creating_api_keys_for_app_store_connect_api

App Store Connect : https://appstoreconnect.apple.com

  1. appstoreconnect.apple.com으로 접속한다.
  2. 사용자 및 액세스 선택

  1. 상단탭에 “Keys”를 선택한하고 + 버튼을 클릭한다.

“Issuer ID”는 w-hive에 인증서 등록시 필요하여 값을 복사해 인증서등록시 사용한다.

  1. API 키 생성을 위해 이름, 액세스 권한을 선택한다.

  • API키 생성후 파일을 다운로드합니다.

  • 키 ID도 whive에 등록시 사용하여 값을 복사해둔다.

2.2.3 인증서 설정

W-Hive는 Android/iOS 빌드 및 배포시 필요한 Singing Key, Profile, p8, API Key를 관리 하고 설정 하는 기능을 제공합니다. 생성한 인증서 및 프로파일, API키들을 W-Hive에 업로드합니다.

Key Setting 시 필수로 필요한 입력 정보입니다.

(1) Key Name : (Android : 키 프로필 이름 / iOS : 인증서 프로필 이름)

(2) 빌드 서버 프로필 ID : 실제 앱을 빌드하는 장비를 구분하기 위한 구분자

(3) OS : 프로필에 추가할 OS 유형

(4) Key Type : Android,iOS 개발/배포 인증서 유형

Key Type -> Build 선택 할 경우

(5) Key Alias : 키를 식별할 수 있는 이름

(6) Key Password : 키에 사용할 보안 비밀번호

(7) Store Password : 키 저장소에 사용할 보안 비밀번호

(8) Key File : 파일 선택시, KeyStore 파일을 선택해야합니다.

Key Type -> Deploy 선택 할 경우

(1)번에 있는 Key Type을 (2) JSON Key File : 파일 선택시 Json Key file를 선택해야합니다.

iOS Profile 설명

Key Type -> Build 선택 할 경우

(1) 개발 Provisiong Profile : 개발 권한 설정 프로파일

(2) 배포유형 : 애플에서 지원하는 배포유형 (app-store, enterprise).

(3) 배포 Provisiong Profile : 배포 권한 설정 프로파일

(4) 애플 개발자 인증서 파일 : 애플 개발자 인증서를 PKCS#12 형식으로 변환한 파일

(5) 애플 개발자 인증서 파일 비밀번호 : 애플 개발자 인증서를 PKCS#12 형식으로 변환시 지정한 비밀번호

Key Type -> Deploy 선택 할 경우

(1) App Store Connect API Key : Apple에서 제공하는 Rest API 서비스 사용시 사용자 인증을 위한 키

(2) ISSUER ID : App Store Connect API Key를 생성한 발급자 식별에 사용되는 ID

(3) KEY ID : App Store Connect API Key를 구분하기 위한 ID

2.3. VCS 설정

        GitHub, Git 정보를 입력하는 설정 화면입니다.

        3.1.2 플러그인 설정

플러그인 설정 화면에서는 화면이동과 동시에 플러그인 리스트 조회가 됩니다.

초기 설정에는 사용가능한 플러그인만 조회가 됩니다.

3.1.2.1. 플러그인 추가

플러그인 추가 할때는 version 을 선택합니다. 그리고 빨간색 박스 안에 보이는 곳에 (+) 버튼을 클릭하면 아래와 같이 적용된 플러그인 리스트로 이동이 됩니다.

적용한 플러그인 설정 적용 완료 하려면 파란색 버튼 완료 를 클릭하면 됩니다.

3.1.2.2. 플러그인 삭제 

적용된 플러그인 리스트를 제거하기 원하시면 빨간색 박스 안에 있는 보라색 (-) 버튼을 클릭하시면 됩니다.

클릭 하시고 나면 맨 하단에 사용가능한 플러그인 리스트로 옮겨지게 된것을 확인 할 수 있습니다.

옮겨지고 나서 파란색 완료 버튼을 누르면 설정 적용 된 것을 확인 할 수 있습니다.

        - 배포 설정

배포 설정

  • iOS 앱스토어 배포 키 : 앱 스토어 배포를 위한 App Store Connect API Key (p8)
  • Android 플레이 스토어 배포키 : 플레이 스토어 배포를 위한 API JSON Key

배포 키를 선택하고 완료 버튼을 누르면 배포 키 설정이 진행되고 완료 됩니다.

3.2. 빌드

빌드는 iOS는 debug/release mode, Android debug/release/AABdebug/AABrelease mode로 구분하여 나눠져 있습니다.

mode 선택 하고 빌드 버튼을 누르면 (git, localgit) pull, build, file upload, app version code up, qrcode url send 기능이 순차적으로 동작합니다.

빌드시 빌드 log 가 화면에 출력 됩니다.

3.3. 빌드 이력

빌드 이력 조회 화면에서는 앱파일 다운로드, 로그파일 다운로드, QRCode, 배포 화면이동 기능을 제공합니다.

3.4. 배포

배포 mode 에 따라 방법이 다릅니다,

alpha, beta, deploy 기본 세가지로 나눠져 있습니다.

사용자가 원하는 배포 mode 설정하고 배포 버튼을 누르면됩니다.

3.5. 배포 이력

배포 한 이후 log 나 결과 조회 가 가능합니다.

해당 화면에서도 log 파일을 다운로드를 제공합니다.
